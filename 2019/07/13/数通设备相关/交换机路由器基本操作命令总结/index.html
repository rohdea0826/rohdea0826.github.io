交换机路由器基本操作命令总结

2019-07-13
Switch, VRP

VRP

通用路由平台VRP（Versatile Routing Platform）是华为公司数据通信产品的通用操作系统平台。
本篇文章将根据以下题目总结一些基本操作

  1. 要求PC1访问PC3在SW1-SW3上实现负载分担，访问PC2经过R1.要求使用ospf实现

int Eth-Trunk 1
quit
int g0/0/2
eth-trunk 1
quit
int g0/0/6
eth-trunk 1
quit

交换机通过vlan配置IP地址

由于交换机工作在TCP/IP协议的第二层数据链路层，其端口也属于二层端口，因此不能直接在该端口上设置三层口的IP地址。
但在交换机上可以通过设置vlan的方式来实现三层口的功能，即vlanif。
vlan，即virtual lan，也就是虚拟局域网的意思。

此处配置若无明确说明均为SW1或R1，其余交换机或路由器配置类似。

设置vlan并配置vlanif的ip地址

设置vlan。
vlan可以设置单个vlan，vlan batch可以同时创建多个vlan。

sys
vlan batch 10 11 113
quit

分别设置vlanif的ip地址。

interface vlanif 10
ip address 1.1.1.254 24
quit

interface vlanif 11
ip address 172.16.11.2 24
quit

interface vlanif 113
ip address 172.16.113.1 24
quit
设置交换机端口的类型并绑定相应的vlan

access连接类型发送数据帧时不带vlan的标签，因此一般连接主机或路由器的三层口。接收数据时会添加default vlan的vlan标签。
trunk连接类型发送数据帧时带有vlan的标签，因此一般与同样是trunk类型的交换机接口相连。即收发的数据帧都带有相应的vlan标签。通过allow-pass vlan设置允许通过的相应数据帧。
hybrid连接类型是华为设备所特有的，通过tagged或untagged可以设置是否发送带有vlan标签的数据帧。因此既可连接主机路由器，也可连接交换机的trunk口。

int g0/0/3
port link-type access
port default vlan 10
quit

int g0/0/1
port link-type hybrid
port hybrid pvid vlan 11
port hybrid untagged vlan 11
quit

int eth-trunk 1
port link-type trunk
port trunk allow-pass vlan 113

添加静态路由

需要实现PC1与PC3的互访。
因此首先需要在SW1上添加静态路由

ip route-static 3.3.3.0 24 172.16.113.2

其次在SW3上添加静态路由以确保PC3返回的ICMP reply报文能够正确返回给PC1

ip route-static 1.1.1.0 24 172.16.113.1

由于需要互访，因此也需要在R1和R2上配置静态路由。
R1:

ip route-static 1.1.1.0 24 172.16.11.2

R2:

ip route-static 2.2.2.0 24 172.16.22.2

路由器ospf配置

ospf 1 router-id 10.1.1.1
area 0
network 172.16.11.0 0.0.0.255
network 172.16.12.0 0.0.0.255
quit
quit

将外部网络的静态路由引入ospf

当PC1向PC2发送数据包时，由于两台路由器ospf中的LSDB中没有目的地址的网段路由，因此需要将设置的静态路由引入。

ospf 1
import-route static

到此，就实现了第一条需求。
